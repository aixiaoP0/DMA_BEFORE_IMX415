## 2. `new/delete` vs `malloc/free`

**对应八股.pdf 页码**：第 28 页
**PDF 核心考点**：malloc 分配堆内存不调用构造函数（仅分配原始字节）；new 分配+构造；free 不调用析构函数

**代码对照** — 旧架构是展示 `malloc/free` 与 `new/delete` 混合使用的典型案例：

| 场景 | 使用方式 | 原因 |
|------|---------|------|
| NV12 数据拷贝 | `malloc(nv12_size)` / `free(nv12_data)` | 原始字节缓冲区，无需构造/析构 |
| DMA buf 数组 | `calloc(num_buffers, sizeof(...))` | C 风格数组，零初始化 |
| YOLO 模型实例 | `make_shared<Yolov5s>(model_path, i%3)` | C++ 对象，需要构造函数 |
| 模型二进制数据 | `(unsigned char*)malloc(size)` / `free(model_data)` | 二进制字节流 |

**关键代码行**：

`main.cpp:251-254` — ReadVideo 线程中 malloc NV12：
```cpp
frame_temp.nv12_data = (uint8_t*)malloc(nv12_size);
if (frame_temp.nv12_data) {
    memcpy(frame_temp.nv12_data, buffer_info.mapped_addr, nv12_size);
}
```

`main.cpp:423` — WriteVideo 线程中 free：
```cpp
free(output_FD.nv12_data);
output_FD.nv12_data = nullptr;  // 置空防悬空指针
```

`thread_pool.cpp:63` — 线程池用 new（通过 shared_ptr）：
```cpp
std::shared_ptr<Yolov5s> yolo = std::make_shared<Yolov5s>(model_path, i%3);
```

> **面试洞察**：旧架构中 NV12 数据用 `malloc`/`free` 是因为它本质是**原始字节流**（1.38MB byte array），不是 C++ 对象，用 new 的 `operator new[]` 会带来**额外 4 字节头部**（记录数组长度），在嵌入式场景中应避免。而 YOLO 模型对象有复杂构造逻辑（NPU 初始化），必须用 new。旧架构的 malloc NV12 正是面试中那个"唯一不可避开的 CPU 拷贝"的路径。

## 4. struct 内存对齐

**对应八股.pdf 页码**：第 33 页
**PDF 核心考点**：自然对齐规则、内存对齐 padding、union 共享内存

```cpp
    int index;               // 4 字节
```

在 64 位系统上：`cv::Mat` 约 16+ 字节 → `int index`(4) → padding(4 字节对齐到 8) → `uint8_t*`(8) → ... 实际大小大于成员总和。

`post_process.h:31-46` — 检测结果结构体（面试常考 struct padding）：
```cpp
typedef struct _box_t {
    int left_top_x;          // 4
    int left_top_y;          // 4
    int right_bottom_x;      // 4
    int right_bottom_y;      // 4   → 共 16 字节，4 字节对齐良好

    // total: 36 字节 → 自然对齐无 padding（16 %4=0, 4%4=0, 16%4=0）

```

> **面试加分**：这里 `detect_result_t` 由于 `char[16]` (16) → `float` (4) → `box_t` (16) 的布局在 32 位系统上天然对齐无 padding，可做 4 字节对齐分析。

## 5. `#define` 宏与位运算

**对应八股.pdf 页码**：第 28-29 页（`#` 和 `##` 预处理器运算符）
**PDF 核心考点**：`#` 字符串化、`##` 记号拼接、宏的副作用问题

**代码对照**：

`main.cpp:60` — 向上取整宏（典型位运算技巧）：
```cpp
#define ALIGN(x, a) (((x)+(a)-1)&~((a)-1))
```
这是嵌入式面试高频题！将 x 按 a（a 必须是 2 的幂）向上对齐：
- `(x)+(a)-1` — 向上取整的加法技巧
- `~((a)-1)` — 生成长度为 `log2(a)` 的低位掩码
- 两者 `&` 操作清除低位

示例：`ALIGN(720, 16) = ((720+15) & ~15) = (735 & 0xFFFFF0) = 720`（720 正好对齐）
`ALIGN(722, 16) = ((722+15) & ~15) = (737 & 0xFFFFF0) = 736`

代码中实际使用（`main.cpp:594-595`）：
```cpp
g_hor_stride = ALIGN(width, 16);   // 1280 → 1280（已对齐）
g_ver_stride = ALIGN(height, 16);  // 720 → 720（已对齐）
```
这里 width=1280 和 height=720 恰好是 16 的倍数，但 ALIGN 依然重要——**RGA 硬件要求 stride 为 16 字节对齐**，如果分辨率变为 1920x1080（1080 不是 16 的倍数），ALIGN 会输出 `1088`。

头文件防卫宏（`thread_pool.h:1-2`）：
```cpp
#ifndef __THREAD_POOL_H
#define __THREAD_POOL_H
```

## 7. `poll`（I/O 多路复用）

**对应八股.pdf 页码**：第 49-50 页（select/poll/epoll 系列）

| 对比项 | select | poll | epoll |

## 14. 条件变量 `condition_variable`（SafeQueue 实现）

**对应八股.pdf 页码**：【补充，PDF 25-66 页未直接覆盖，属于面试高频考点】
**经典八股考点**：虚假唤醒、predicate wait、unique_lock

`SafeQueue.h:21-44` — 双条件变量有界队列（面试必问题）：
```cpp
void enqueue(const T& t) {
    unique_lock<mutex> lock(m);
    cond_not_full.wait(lock, [this]{return q.size() < maxSize || stop_flag;});  // predicate wait
    q.push(t);
    cond_not_empty.notify_all();
}

bool dequeue(T& t) {
    unique_lock<mutex> lock(m);
    cond_not_empty.wait(lock, [this]{return !q.empty() || stop_flag;});
    if (stop_flag && q.empty()) return false;
    t = q.front(); q.pop();
    cond_not_full.notify_all();
    return true;
}
```

| 八股考点 | 代码实现 |
|---------|---------|
| 虚假唤醒 | `wait()` 的 predicate 重载（第二个参数）自动处理 |
| 双条件变量 | 满条件 `cond_not_full` + 空条件 `cond_not_empty`，避免"生产者唤醒生产者" |
| 超时保护 | 无（生产-消费模式不需要，5 秒无帧通过 poll 超时处理） |
| 安全退出 | `stop_flag` 确保停止时所有线程都能被唤醒 |

## 15. 线程绑核（CPU Affinity）

**对应八股.pdf 页码**：【补充，PDF 25-66 页未直接覆盖，属于操作系统经典话题】


## 18. `typedef` vs `#define`

**对应八股.pdf 页码**：第 44 页
**PDF 核心考点**：`typedef` 有类型检查、`#define` 纯文本替换、`typedef int* PINT` vs `#define PINT int*` 的区别

`thread_pool.h:31` — 结构体 typedef：

```cpp
typedef struct ProcessResult {
    cv::Mat processed_img;
```

`post_process.h:31-36`：

```cpp
typedef struct _box_t {
```

在 C++ 中声明 `struct` 时可以省略 `typedef`（`struct box_t {...};` 即可直接使用 `box_t`），但保留 `typedef` 是 C 兼容风格，对嵌入式面试**加分**——表明你理解 C/C++ 混合开发场景。

## 19. 短路求值（Short-circuit Evaluation）

**对应八股.pdf 页码**：第 34 页
**PDF 核心考点**：`||` / `&&` 短路特性

虽然旧架构代码没有直接使用 `||`/`&&` 短路求值的经典面试题，但类似思想体现在条件判断的效率优化中：

```cpp
task_cond.wait(lock, [this]{return (!tasks.empty() || !run_flag);});
```
