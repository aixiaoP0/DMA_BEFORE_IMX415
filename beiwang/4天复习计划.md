## Day 1 — Pipeline 数据流骨架（建立全局认知）

**目标**：能徒手画出完整数据流图，理解每个阶段干什么

### 学习内容

**上午（2h）：**

1. **数据流复习目录.md** → 先看那张 ASCII 架构图，记住 8 个阶段的**顺序**和**箭头方向**

2. **数据流复习_V4L2+DMA.md**（阶段 1-2）：
   - 记住：IMX415 → MIPI → ISP → NV12 → DMABUF → V4L2 DQBUF
   - 关键数字：4 个缓冲区、5 秒 poll 超时、1280×720 NV12 大小 1.38MB
   - **面试关键词**：MPLANE、DMABUF fd、RKAIQ、CMA 预分配

**下午（2h）：**

3. **数据流复习_RGA转换.md**（阶段 3）：
   - Path A（NV12→BGR 画框）— `importbuffer_fd` → `imcvtcolor`
   - Path C（NV12→RGB→NPU fd 直写）— 旧架构中这是 Path B malloc+memcpy
   - **面试关键词**：importbuffer_fd、importbuffer_virtualaddr、imcvtcolor、imresize

4. **数据流复习_Aggregator调度保序.md**（阶段 4）：
   - 旧架构：`future<ProcessResult>` + `future.get()` 阻塞等待 → head-of-line blocking
   - **面试关键词**：round-robin、per-worker 队列、result_queue、nextWriteIndex

**晚上（1.5h）：**

5. **数据流复习_Worker线程池推理.md**（阶段 5）：
   - NV12→NPU 输入 → rknn_run → 反量化+NMS → 画框 → BGR→NV12
   - **面试关键词**：rknn_create_mem、virt_addr、fd、NMS、INT8 反量化

6. **数据流复习_MPP编码与RTMP推流.md** + **数据流复习_资源释放与异常处理.md**（阶段 6-8）：
   - **面试关键词**：MPP H.264、H.264→AVCC、RTMP、谁分配谁释放

**睡前复习（15min）**：默画数据流图，标注 Zero Copy 路径上的 `importbuffer_fd` 调用点

## Day 2 — 线程模型 + 零拷贝核心（攻克最难点）

**目标**：能清晰解释 SafeQueue 双条件变量、ThreadPool 调度、零拷贝的三方共享同一块内存

### 学习内容

**上午（2h）：**

1. **线程同步_SafeQueue.md**：
   - 核心机制：双条件变量设计（`cond_not_full` + `cond_not_empty`）
   - predicate wait 处理虚假唤醒
   - `stop_flag` 优雅退出
   - **对着代码逐行看**：SafeQueue.h:21-44，理解 `wait(lock, [this]{...})` 的 lambda 条件

2. **线程池_ThreadPool.md**：
   - 新架构 vs 旧架构对比表
   - 新架构：`submit_to_worker()` + per-worker 队列 + `try_get_result()` + `result_buffer map`
   - 旧架构：`submit_task_async()` + `future<ProcessResult>` + 全局 SharedQueue
   - **面试重点**：旧架构的 future.get() 阻塞 vs 新架构的 try_get_result() 非阻塞 + map 缓存 = 核心演进思路

**下午（2h）：**

3. **零拷贝数据通路重构.md**：
   - 从旧架构的 Path B `malloc+memcpy` 到新架构 Path C `RGA 直写 NPU fd`
   - 理解为什么 DMABUF 必须 QBUF 归还导致需要拷贝
   - **面试必问**：旧架构那道"唯一不可避免的 CPU 拷贝"

4. **YOLOv5s零拷贝内存绑定.md**：
   - `rknn_create_mem` 分配的 `rknn_tensor_mem` 结构体：
     ```
     .virt_addr → CPU 读输出（mmap）
     .fd        → RGA 写输入（importbuffer_fd）
     .logical_addr → NPU 推理用
     ```
   - **面试核心**：同一块物理内存，三方通过不同接口访问，零拷贝

**晚上（1.5h）：**

5. **面试_零拷贝数据通路.md** — 直接背诵 Q1 的回答
6. **面试_CMA池与缺页中断.md** — 理解缺页中断的 perf 统计、Glibc 动态阈值机制
7. **面试_NPU初始化位置设计决策.md** — RAII 封装、高内聚低耦合的设计理由

**睡前复习（15min）**：自己口述"rknn_tensor_mem 零拷贝机制"的完整解释

## 学习原则

1. **不需要逐字背诵**——面试考察**理解后用自己的话表达**
2. **每个文档只记 3 个面试关键词 + 1 个核心原理**
3. **睡前必做** "白纸默画数据流图"（Day 1-4 都重复）
4. **面试问题.md 不要全背**，要理解回答思路的逻辑链条（问题 → 解决方案 → 为什么选择这个方案）
5. **新旧架构对比是核心区分度**— 面试中能讲出 Path B→Path C 的演进理由，比只讲当前实现更有价值
